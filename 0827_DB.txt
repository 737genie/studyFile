0827_DB

1. 데이터베이스 
 - 저장, 연결, 조회 등의 각종 데이터 처리
 - DB는 미들웨어로 관리
 - DB 관리하는 미들웨어 = DBMS
   ex. oracle, my-sql, mariaDB, ms-sql, postgreSQL, cybase, tibero ...


1) DB에서 데이터를 분류하는 기준 - key, column 등 (CRUD 방식)
2) 시나리오
- 경비견 누렁이의 데이터 창고(DB) 경비시스템 가이드
- 누렁이는 DB 전체를 지킴

- 한 건물 안에는 여러 개의 방들(테이블)이 있음

- 각 방 별로 특정한 종류의 물건들만 보관

- 각 방 별로 준비되어 있는 수납장(column)이 있음
	- 수납장에는 특정한 종류의 정보만 넣음


열 = column, 필드, attribute
행 = 튜플, 레코드


2. SQL - DB에서 데이터를 정의, 조작, 제어하기 위해 쓰는 언어

1) DDL : Data Definition Language 
-> 데이터 정의
-> DB, 테이블을 생성 삭제하거나 그 구조를 변경할 수 있음
(CREATE, ALTER, DROP;현업가서는 최대한 쓰지말 것)

2) DML : Data Manipulation Language
-> 데이터 조작
-> DB에 저장된 데이터를 처리, 조회, 검색할 수 있음
(select, insert, update, delete)

3) DCL : Data Control Language
-> 데이터 제어
-> DB에 저장된 데이터를 관리, 데이터 무결성 제어 할 수 있음
(Grant, REVOKE)


* 무결성 : 정보에 결점이 없도록 유지하는 성질 - DB는 기본적으로 무결성을 지키는 것이 최우선



3. DB 용어

- 데이터베이스 : 테이블들이 모여있는 공간
- 스키마 : 데이터베이스의 구조, 설계도

- 무결성을 위한 제약조건

-- PRIMARY KEY : 기본키 설정
-- NOT NULL : null값 xx
-- UNIQUE : 테이블들의 row 값은 중복 되어선 안 됨(null은 1개는 허용)
-- DEFAULT : 기본값 설정 (null값 방지)
-- AUTO_INCREMENT : 자동증가 (기본은 숫자 1부터 시작해서 1씩 증가)
-- FOREIGN KEY : 외래키 설정
